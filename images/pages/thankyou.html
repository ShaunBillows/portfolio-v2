Portify

introduction / Problem Statement

As an avid user of Spotify I was always dispointed by its genre filtering capabilities, particularly when it came to dance music. I set out to create an application to solve this problem by combining the functionality of Spotify and Beatport, an online electronic music store which classifies it's electronic music collection into broad, discrete genre categories, unlike Spotify.

Methodology / Implementation

Portify consists of two decoupled applications written in diffent programming languages; a web scraper written in Python for collecting and processing the relevant tracks information from Spotify and Beatport and an interface for displaying and interacting with the data, created using the MERN tech stack.

The web scraper connects to Spotify through their API and requests the tracks information about a playlist. For each track in the playlist the corresponding Beatport url is retrieved by scraping and sorting the urls from a Google search using the Requests-HTML library. The Beatport genre associated with each url is then extracted by parsing the HTML using the Beautiful Soup library and selecting the genre by CSS selector . Upon completion the data is loaded into a MongoDB database. Python was chosen for this part of the project for its superior web scraping and data mining capabilities.

To interact with the data a REST API was created using Express.js which pulled the data into a React app, which consisted of an interface. One major problem with web scraping is that data collected is rarely 100% accurate, and in this case things are no different. From experience, around 5% of the tracks on Spotify were not on Beatport and another 5% returned an incorrect genre. The later was largely due to flaws in the url selection algorithm targetting the data scraped from Google, which contained several, infrequent corner cases which are not accounted for. To solve this problem Portify's interface allows for the Beatport url from which the track's genre was sourced to analyed in order to highlight any discrepencies, which can then be corrected for manually. This mechanism conviniently allows for playlists to be cusomised and altered by taste as well.

The Portify interface supports full CRUD functionality. Playlists can edited, copied, mergered, deleted and, importantly, sorted by genre. Since the Spotify uri for each track is saved, Portify playlists can be uploaded back to Spotify on the click of a button.

Conclusion

Portify allows Spotify playlists to be sorted accurately into sub playlists based on their Beatport genre and uploaded back to Spotify. Additionally playlists are stored in a database where they can be edited, copied, merged and deleted using a custom-made UI.
